# Skill Benchmark 评分细则

## 五维评估标准

### 1. 触发准确率 (Trigger Accuracy) — 权重 25%

| 分数 | 标准 |
|------|------|
| 100% | 所有正向用例触发，所有负向用例跳过 |
| 90%  | 1 个误判（漏触发或误触发） |
| 80%  | 2 个误判 |
| 70%  | 3 个误判 |
| <60% | 描述需要重写 |

**评估要点**:
- description 是否包含足够的触发关键词
- 是否有"推性"描述防止欠触发
- 负向场景是否与相邻 Skill 区分清晰

### 2. 输出质量 (Output Quality) — 权重 25%

| 分数 | 标准 |
|------|------|
| 100% | 指令完整、有示例、格式定义清晰、覆盖所有场景 |
| 90%  | 指令完整、有示例、格式基本清晰 |
| 80%  | 指令完整但缺少示例或格式定义 |
| 70%  | 覆盖主要场景但有遗漏 |
| <60% | 指令模糊或严重不完整 |

**检查项**:
- [ ] 是否有明确的输出格式定义
- [ ] 是否包含至少 1 个完整示例
- [ ] 是否覆盖正常流程和错误流程
- [ ] 指令是否使用祈使句（而非被动语态）
- [ ] 是否解释了 "why" 而非只有 "what"

### 3. 工具使用 (Tool Usage) — 权重 20%

| 分数 | 标准 |
|------|------|
| 100% | allowed-tools 精确匹配需求，指令中明确引导工具选择 |
| 90%  | 工具列表合理，有基本引导 |
| 80%  | 工具列表偏宽或偏窄 |
| 70%  | 缺少工具使用指导 |
| <60% | 工具列表与功能不匹配 |

**检查项**:
- [ ] allowed-tools 是否是最小必要集
- [ ] 是否有工具使用的优先级指导
- [ ] 是否避免了不安全的工具使用模式

### 4. 错误处理 (Error Handling) — 权重 15%

| 分数 | 标准 |
|------|------|
| 100% | 覆盖所有常见错误、有降级策略、错误信息可操作 |
| 90%  | 覆盖主要错误路径 |
| 80%  | 有基本错误处理但不全面 |
| 70%  | 仅处理了最明显的错误 |
| <60% | 无错误处理或处理不当 |

**常见错误场景**:
- CLI 未安装 / 版本不兼容
- 集群连接失败
- 配置文件缺失
- 权限不足
- 网络超时

### 5. Token 效率 (Token Efficiency) — 权重 15%

| 分数 | 标准 |
|------|------|
| 100% | SKILL.md < 200 行，信息密度高，references 按需加载 |
| 90%  | SKILL.md < 300 行，结构合理 |
| 80%  | SKILL.md < 400 行，有一些冗余 |
| 70%  | SKILL.md < 500 行，需要精简 |
| <60% | SKILL.md > 500 行，或无分层加载 |

**计算方法**:
```
skill_tokens = wc -w SKILL.md (近似 token 数)
ref_tokens = wc -w references/*.md (总引用 token)
功能点数 = 手动计数的独立功能/场景数
信息密度 = 功能点数 / (skill_tokens / 100)
```

| 信息密度 | 评价 |
|----------|------|
| > 3.0    | 优秀 |
| 2.0-3.0  | 良好 |
| 1.0-2.0  | 一般 |
| < 1.0    | 需精简 |

---

## 综合评分计算

```
overall = trigger_accuracy * 0.25
        + output_quality * 0.25
        + tool_usage * 0.20
        + error_handling * 0.15
        + token_efficiency * 0.15
```

## 回归判定阈值

| 维度 | 允许下降幅度 | 阻断性回归 |
|------|-------------|-----------|
| 触发准确率 | -5% | -15% |
| 输出质量 | -10% | -20% |
| 工具使用 | -5% | -15% |
| 错误处理 | -10% | -20% |
| Token 效率 | +20% (增长) | +50% |

**阻断性回归**: 任一维度超过阻断阈值则标记为 BLOCKED，不允许合并。
