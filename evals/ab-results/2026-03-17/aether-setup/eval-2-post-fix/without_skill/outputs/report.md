|------|------|------|

| `http://192.168.69.70:4646` | **200 OK** | infra-server-1 (follower) |
| `http://192.168.69.71:4646` | **200 OK** | infra-server-2 (当前 leader) |
| `http://192.168.69.70:8500` | **200 OK** | Consul 服务正常 |

### 集群节点 IP 分布

Aether 集群的 IP 范围如下：

- **Nomad Server**: `192.168.69.70` - `192.168.69.72`
- **Heavy Clients**: `192.168.69.80` - `192.168.69.82`
- **Light Clients**: `192.168.69.90` - `192.168.69.94`

`192.168.69.99` 不在上述任何范围内。

### 正确的 Nomad 地址

正确的 Nomad API 地址为 `http://192.168.69.70:4646`（全局配置中已正确设置）。

```yaml
  nomad_addr: "http://192.168.69.70:4646"
  consul_addr: "http://192.168.69.70:8500"

  registry_url: "forgejo.10cg.pub"
```

**状态**: 全局配置正确。Nomad 和 Consul 地址均指向正确的集群节点，连通性已验证。

### 项目级配置 (`/home/dev/Aether/.aether/config.yaml`)

```yaml

  registry: ghcr.io
  registry_url: ghcr.io
```

**状态**: 项目级配置已存在，且 registry 已设置为 `ghcr.io`。根据 Aether 的配置优先级（项目配置 > 全局配置），该项目使用时会覆盖全局的 `forgejo.10cg.pub` registry 设置。

### 环境变量检查

未发现 `NOMAD_ADDR`、`CONSUL_ADDR` 或其他相关环境变量覆盖，排除环境变量干扰。

## 3. 配置优先级确认

Aether 配置优先级为：

```
环境变量 > 项目配置 > 全局配置 > 默认值
```

对于当前项目：

| 配置项 | 生效值 | 来源 |
|--------|--------|------|
| `nomad_addr` | `http://192.168.69.70:4646` | 全局配置 |
| `consul_addr` | `http://192.168.69.70:8500` | 全局配置 |
| `registry` | `ghcr.io` | 项目配置 (覆盖全局) |
| `registry_url` | `ghcr.io` | 项目配置 (覆盖全局) |

## 4. 结论与建议

### 问题根因

用户使用的 `http://192.168.69.99:4646` 地址不正确。该 IP 不属于集群中任何节点。正确的 Nomad 地址是 `http://192.168.69.70:4646`，全局配置中已正确设置。

1. **全局配置** (`~/.aether/config.yaml`): 正确，无需修改。
2. **项目级配置** (`/home/dev/Aether/.aether/config.yaml`): 已存在且已设置 `ghcr.io` 作为 registry，满足需求。
3. **环境变量**: 无覆盖，无问题。

- 使用 Aether CLI 时无需额外操作，配置已正确。请确保不要在命令行中手动指定 `192.168.69.99` 地址。
