确认方案后，以下是为 user-service 生成的完整部署文件。

### 文件 1: Dockerfile

**路径**: `user-service/Dockerfile`

```dockerfile
# Generated by Aether
# Project: user-service
# Language: Go

FROM golang:1.22-alpine AS builder

WORKDIR /app

# Install build dependencies
RUN apk add --no-cache git ca-certificates

# Copy dependency files first (layer caching)
COPY go.mod go.sum* ./
RUN go mod download

# Copy source code
COPY . .

# Build the application
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -ldflags="-w -s" -o main .

# Final stage - minimal image
FROM alpine:latest

RUN apk --no-cache add ca-certificates tzdata

WORKDIR /root/

# Copy binary from builder
COPY --from=builder /app/main .
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

EXPOSE 8080

CMD ["./main"]
```

**说明**:
- 多阶段构建：builder 阶段编译，final 阶段仅包含二进制
- `CGO_ENABLED=0` 生成静态链接二进制
- `-ldflags="-w -s"` 去除调试信息，减小镜像体积
- `go.sum*` 使用通配符以兼容尚无 go.sum 的新项目
- 最终镜像基于 alpine，体积约 10-15 MB

### 文件 2: deploy/nomad-dev.hcl

**路径**: `user-service/deploy/nomad-dev.hcl`

```hcl
# Generated by Aether
# Project: user-service
# Environment: dev

job "user-service-dev" {
  datacenters = ["dc1"]
  type = "service"

  update {
    max_parallel     = 1
    min_healthy_time = "10s"
    healthy_deadline = "3m"
    progress_deadline = "10m"
    auto_revert      = true
  }

  group "user-service" {
    count = 1

    network {
      port "http" {
        static = 8080
      }
    }

    service {
      name = "user-service"
      tags = ["dev", "user-service"]
      port = "http"

      check {
        type     = "tcp"
        interval = "10s"
        timeout  = "2s"
      }
    }

    task "user-service" {
      driver = "docker"

      config {
        image = "forgejo.10cg.pub/user-service:dev"
        ports = ["http"]
      }

      resources {
        cpu    = 500
        memory = 512
      }
    }
  }
}
```

**说明**:
- 单副本，static port 8080 用于开发调试
- TCP 健康检查每 10 秒
- `auto_revert = true` 部署失败自动回滚
- 运行在 dc1 数据中心的 heavy nodes

### 文件 3: deploy/nomad-prod.hcl

**路径**: `user-service/deploy/nomad-prod.hcl`

```hcl
# Generated by Aether
# Project: user-service
# Environment: prod

job "user-service-prod" {
  datacenters = ["dc1"]
  type = "service"

  update {
    max_parallel      = 1
    min_healthy_time  = "30s"
    healthy_deadline  = "5m"
    progress_deadline = "15m"
    auto_revert       = true
    auto_promote      = true
  }

  group "user-service" {
    count = 2

    network {
      port "http" {
        to = 8080
      }
    }

    service {
      name = "user-service"
      tags = ["prod", "user-service", "urlprefix-/user-service"]
      port = "http"

      check {
        type     = "tcp"
        interval = "15s"
        timeout  = "3s"
      }

      meta {
        version = "latest"
      }
    }

    task "user-service" {
      driver = "docker"

      config {
        image      = "forgejo.10cg.pub/user-service:latest"
        ports      = ["http"]
        force_pull = true
      }

      resources {
        cpu    = 500
        memory = 512
      }
    }
  }
}
```

**说明**:
- 2 副本，动态端口映射到容器 8080
- 更严格的健康检查（30s min_healthy_time, 5m deadline）
- `urlprefix-/user-service` 标签用于 Fabio/Traefik 路由
- `force_pull = true` 确保始终拉取最新镜像
- `auto_promote = true` 自动提升 canary

### 文件 4: .forgejo/workflows/deploy.yaml

**路径**: `user-service/.forgejo/workflows/deploy.yaml`

```yaml
# Generated by Aether
# CI/CD workflow for deploying user-service
# Registry: forgejo.10cg.pub (Forgejo)

name: Deploy user-service

on:
  push:
    branches:
      - main
      - master
  pull_request:
    branches:
      - main
      - master
  workflow_dispatch:

env:
  REGISTRY: forgejo.10cg.pub
  PROJECT: user-service
  PORT: 8080

jobs:
  build:
    name: Build Docker Image
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Forgejo registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ secrets.FORGEJO_USER || secrets.REGISTRY_USERNAME }}
          password: ${{ secrets.FORGEJO_TOKEN || secrets.REGISTRY_PASSWORD }}

      - name: Determine tag
        id: tag
        run: |
          if [[ "${{ github.ref }}" == refs/tags/v* ]]; then
            echo "tag=${GITHUB_REF#refs/tags/}" >> $GITHUB_OUTPUT
            echo "latest=true" >> $GITHUB_OUTPUT
          else
            echo "tag=dev" >> $GITHUB_OUTPUT
            echo "latest=false" >> $GITHUB_OUTPUT
          fi

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: ${{ github.event_name != 'pull_request' }}
          tags: |
            ${{ env.REGISTRY }}/${{ env.PROJECT }}:${{ steps.tag.outputs.tag }}
            ${{ steps.tag.outputs.latest == 'true' && format('{0}/{1}:latest', env.REGISTRY, env.PROJECT) || '' }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy-dev:
    name: Deploy to Nomad Dev
    runs-on: ubuntu-latest
    needs: build
    if: github.ref == 'refs/heads/main' || github.ref == 'refs/heads/master'
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Deploy to Nomad
        run: |
          curl -s -X POST \
            -H "X-Nomad-Token: ${{ secrets.NOMAD_DEV_TOKEN }}" \
            -d @deploy/nomad-dev.hcl \
            ${{ secrets.NOMAD_DEV_ADDR }}/v1/jobs/parse || true

          # Alternative: use nomad CLI directly
          export NOMAD_ADDR="${{ secrets.NOMAD_DEV_ADDR }}"
          export NOMAD_TOKEN="${{ secrets.NOMAD_DEV_TOKEN }}"
          nomad job run deploy/nomad-dev.hcl

      - name: Wait for deployment
        run: |
          echo "Waiting for deployment to stabilize..."
          sleep 30

      - name: Health check
        run: |
          for i in {1..5}; do
            if curl -sf http://dev.user-service.internal:8080/health 2>/dev/null; then
              echo "Health check passed!"
              exit 0
            fi
            echo "Attempt $i/5 - retrying in 10s..."
            sleep 10
          done
          echo "Warning: Health check endpoint not reachable (service may still be starting)"
```

**说明**:
- Forgejo registry 凭据：使用 `FORGEJO_USER`/`FORGEJO_TOKEN`，回退到通用 `REGISTRY_USERNAME`/`REGISTRY_PASSWORD`
- PR 时只构建不推送镜像（`push: ${{ github.event_name != 'pull_request' }}`）
- 仅 main/master 分支推送时触发部署
- Tag 推送时同时打 latest 标签

## 注意事项

1. **go.sum 文件**: 确保在 `go mod tidy` 后 `go.sum` 文件存在，Dockerfile 中的 `COPY go.mod go.sum* ./` 使用通配符兼容两种情况。

2. **健康检查端点**: 建议在 `main.go` 中添加 `/health` 端点，用于 CI 和 Nomad 健康检查：
   ```go
   http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
       w.WriteHeader(http.StatusOK)
       w.Write([]byte("ok"))
   })
   ```

3. **端口冲突**: dev 环境使用 static port 8080，如果有多个服务使用相同端口，需要修改为不同端口或使用动态端口。

4. **资源配额**: 默认分配 500 MHz CPU + 512 MB 内存，根据实际负载调整。

5. **Consul 服务注册**: Nomad job 中的 `service` 块会自动注册到 Consul，可通过 `user-service.service.consul` DNS 访问。

6. **生产部署**: 生产环境使用 `deploy/nomad-prod.hcl`，通过 Git tag (v*) 触发，包含 canary 部署策略。
